# アプリ概要

偉人の名言をクールに表示するサイト
スタイリッシュで見ていてモチベーションが上がる感じ
ランダム名言表示、ピン留めした名言の表示、名言登録の機能がある

# 機能概要

- 今日の格言: ダークトーンの背景に、ランダムに表示される名言と偉人の情報が際立つスタイリッシュなメイン画面です。お気に入りとしてピン留めするボタンも配置されています。
- お気に入りの格言: ピン留めした名言がシンプルで洗練されたカード形式で一覧表示される画面です。各カードからは名言の編集や削除が可能です。
- 名言を登録: 新しい名言と作者を登録するためのフォーム画面です。モダンな入力フィールドと視認性の高い登録ボタンで、操作性を重視したデザインになっています。

# 機能詳細：今日の格言

ランダムで 1 個格言を表示する
格言は同じフォルダにある quote.txt にあるので、そこから格言の JSON ファイルを作る
発言した人の名前がないので適当に考えて設定しておく
格言には背景画像をランダムで表示する
背景画像素材は同じフォルダにある image1.webp,image2.webp,image3.webp を使用する
Pin のボタンを押すと格言をピン留めでき、「お気に入りの格言」画面で表示できる
Pin 留め情報は localStorage に保存する

# 機能詳細：お気に入りの格言

Pin 留めされた格言をカードで表示する
Pin 留めを解除する機能はバックログ

# 機能詳細：名言を登録

格言、発言者を登録でき、お気に入りの格言で表示できる
背景は素材からランダムで選ばれる

# 機能詳細：UI

登録されている figma mcp から AI_DEV プロジェクトのデザインを参照

# 要確認事項への回答

- quote.txt→JSON 化をどのタイミング・方法で行い、そのファイルをどこに配置するか（ビルド時に固定 JSON を用意する想定か、起動時に変換するスクリプトを用意するか）を明示したいです。

回答：開発時に JSON 化しておいて、public フォルダに入れてアプリ起動後に動的に fectch してください

- ランダム表示に自作登録の名言も含めるか、初期セットのみを対象にするかを決めたいです。

回答：自作登録の名言も含めてください

- 新規登録した名言をどこに保存するか（localStorage で良いか、キー名や構造、永続化の単位など）を確認したいです。

回答：localStorage に保存してください。構造はお任せしますが、名言と発言者名と何かしらの ID は必須です。

- 「お気に入りの格言」画面はピン留めのみを表示するのか、登録済みの全名言も表示するのかが曖昧です。登録直後にピン留め扱いにするのか、別アクションが必要かを決めたいです。

回答：登録済みの全名言も表示して下さい。登録直後にピン留め扱いで良いです。

- ピン留め済み・登録済みデータの重複防止ルールやバリデーション（空文字、長文など）の要件があれば共有をお願いしたいです。

回答：空文字のチェックのみで良いです。
